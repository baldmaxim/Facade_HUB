# Настройка статусов объектов

## Что было добавлено

Добавлена система статусов для объектов (Тендер, Объекты СУ-10, Проиграли).

## Инструкция по настройке

### 1. Выполнить SQL миграцию

Выполните SQL миграцию для создания таблицы статусов объектов:

```bash
# Откройте Supabase SQL Editor
# Скопируйте и выполните содержимое файла:
supabase/create_object_status_table.sql
```

Эта миграция:
- Создаст таблицу `object_status` со статусами
- Добавит связь с таблицей `objects`
- Создаст начальные статусы: Тендер, Объекты СУ-10, Проиграли

### 2. Управление статусами

В админ-панели (вкладка "Управление") появилась новая карточка "Статусы объектов" 🏢

Здесь вы можете:
- Добавлять новые статусы
- Редактировать существующие
- Удалять ненужные

### 3. Использование

#### При создании объекта
- В модальном окне создания объекта появилось поле "Статус объекта"
- Выберите нужный статус из выпадающего списка

#### При редактировании объекта
- На странице объекта нажмите кнопку редактирования (карандаш)
- В форме редактирования можно изменить статус

#### Отображение статуса
- Статус отображается в профиле объекта в разделе деталей (если установлен)
- Рядом с адресом и застройщиком

## Структура таблиц

### object_status
```
id          UUID PRIMARY KEY
name        TEXT NOT NULL UNIQUE
created_at  TIMESTAMP WITH TIME ZONE
```

### objects (обновлено)
Добавлено поле:
```
status_id   UUID REFERENCES object_status(id)
```

## API функции

Новый файл `src/api/objectStatus.js`:
- `fetchAllObjectStatuses()` - получить все статусы
- `createObjectStatus(name)` - создать статус
- `updateObjectStatus(id, name)` - обновить статус
- `deleteObjectStatus(id)` - удалить статус

Обновлен файл `src/api/objects.js`:
- Функции `fetchObjects()` и `fetchObjectById()` теперь загружают статус
- Добавлена функция `updateObject(id, data)` для обновления объекта
