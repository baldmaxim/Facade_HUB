# Обновление страницы задач

## Изменения

### 1. Структура таблицы в базе данных

Выполните SQL миграцию из файла `supabase_migration_tasks_update.sql` в Supabase SQL Editor.

Миграция создает:
- Таблицу `team_members` (члены команды)
- Добавляет новые столбцы в таблицу `tasks`:
  - `order_number` - порядковый номер задачи
  - `is_high_priority` - флаг высокой приоритетности (красный флажок)
  - `responsible_id` - ID ответственного (связь с team_members)
  - `deadline` - дедлайн выполнения задачи
  - `is_completed` - флаг выполнения задачи
- Удаляет старые столбцы `responsible_zinin`, `responsible_kovalenko`, `responsible_kuznetsov`

### 2. Новая структура формы задач

#### Добавлено:
1. **№** - Порядковый номер задачи (автоматически)
2. **🚩** - Красный флажок для отметки высокоприоритетных задач
3. **Ответственный** - Выпадающий список членов команды из таблицы `team_members`
   - Каждый член команды имеет свой цвет (отображается как левая граница у select)
4. **Дата постановки** - Автоматически заполняется при создании (`created_at`)
5. **Дедлайн** - Поле для выбора даты дедлайна
6. **Выполнено** - Единственный чекбокс для отметки выполнения задачи

#### Удалено:
- Множественные чекбоксы для участников (Зинин, Коваленко, Кузнецов)
- Подзаголовок с именами участников

### 3. Визуальные улучшения

- Выполненные задачи отображаются с пониженной прозрачностью (opacity: 0.6)
- Текст выполненных задач зачеркнут
- Красный флажок (🚩) отображается серым и неактивным по умолчанию
- При установке флага высокого приоритета - флажок становится красным
- У dropdown "Ответственный" есть цветная левая граница по цвету члена команды

### 4. Члены команды по умолчанию

В таблицу `team_members` автоматически добавляются:
- Вячеслав Зинин (синий - #3b82f6)
- Валерий Коваленко (зеленый - #10b981)
- Никита Кузнецов (фиолетовый - #8b5cf6)

### 5. Индексы для производительности

Созданы индексы для оптимизации запросов:
- `idx_tasks_object_order` - по object_id и order_number
- `idx_tasks_responsible` - по responsible_id
- `idx_tasks_deadline` - по deadline

## Как применить изменения

1. Откройте Supabase SQL Editor
2. Скопируйте содержимое файла `supabase_migration_tasks_update.sql`
3. Вставьте в SQL Editor и выполните
4. Обновите страницу в браузере

## Структура данных задачи

```javascript
{
  id: UUID,
  object_id: UUID,
  title: TEXT,
  order_number: INTEGER,
  is_high_priority: BOOLEAN,
  responsible_id: UUID,  // ссылка на team_members
  created_at: TIMESTAMP,
  deadline: DATE,
  note: TEXT,
  is_completed: BOOLEAN
}
```

## API изменения

Теперь при создании задачи используется:
```javascript
{
  object_id: id,
  title: 'текст задачи',
  responsible_id: 'uuid-члена-команды' или null,
  is_high_priority: true/false,
  deadline: '2024-12-31' или null,
  note: 'примечание' или null,
  order_number: автоматически,
  is_completed: false
}
```

При загрузке задач используется join с team_members:
```javascript
.select('*, team_member:responsible_id(id, name, color)')
```
