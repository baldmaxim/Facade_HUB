# Статусы задач

## Описание функционала

Добавлена возможность управления статусами задач:
- **Не начата** - задача создана, но работа еще не началась
- **В процессе** - задача выполняется
- **Завершена** - задача полностью выполнена
- **Отложена** - задача временно приостановлена

## Миграции

### 012_create_task_statuses.sql
Создает таблицу `task_statuses` для хранения статусов задач и добавляет столбец `status_id` в таблицу `tasks`.

## Изменения в коде

### AdminPage.jsx
- Добавлен новый раздел "Статусы задач" в админ-панель
- Возможность создавать, редактировать и удалять статусы задач
- Модальное окно для управления статусами

## Структура таблицы task_statuses

| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Уникальный идентификатор статуса |
| status | TEXT | Название статуса (уникальное) |
| created_at | TIMESTAMP | Дата и время создания записи |

## Изменения в таблице tasks

Добавлен столбец:
- `status_id` (UUID, nullable) - ссылка на статус задачи из таблицы task_statuses

## Как применить

### 1. Выполните миграцию в Supabase

```sql
-- Скопируйте и выполните содержимое файла:
-- 012_create_task_statuses.sql
```

### 2. Код уже обновлен

Все изменения в коде уже применены:
- ✅ AdminPage.jsx (добавлена карточка и модальное окно для управления статусами)

### 3. Проверка

1. Откройте админ-панель
2. Найдите карточку "Статусы задач"
3. Нажмите на карточку
4. Добавьте, отредактируйте или удалите статусы задач
5. Убедитесь, что изменения сохраняются

## Статусы по умолчанию

При применении миграции автоматически создаются следующие статусы:
- Не начата
- В процессе
- Завершена
- Отложена

Вы можете изменить эти статусы или добавить свои через админ-панель.
