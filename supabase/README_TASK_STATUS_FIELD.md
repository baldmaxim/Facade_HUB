# Добавление поля статуса в таблицу задач

## Описание изменений

Добавлено поле "Статус" в таблицу задач с визуальной индикацией статуса цветом:
- **В процессе** - синий цвет (#3b82f6)
- **Отложена** - желтый цвет (#eab308)
- **Завершена** - зеленый цвет (#10b981)
- **Не начата** - серый цвет по умолчанию (#6b7280)

## Изменения в коде

### TasksPage.jsx
1. Добавлен state для статусов задач (`taskStatuses`)
2. Загрузка статусов из таблицы `task_statuses` при монтировании компонента
3. Добавлена колонка "Статус" в таблицу задач
4. Выпадающий список для выбора статуса с цветовой индикацией
5. Функция `getStatusColor()` для определения цвета статуса
6. Обновление `status_id` сохраняется в базу данных с дебаунсом (500мс)

### TasksPage.css
1. Добавлены стили для колонки `.col-status`
2. Добавлены стили для `.status-select` с цветовой границей и фоном

## Структура данных

### Поле в таблице tasks
- `status_id` (UUID, nullable) - ссылка на статус из таблицы `task_statuses`

### Join запрос
```javascript
.select('*, team_member:team_member_id(id, name, role, color), task_status:status_id(id, status)')
```

## Визуальная индикация

Статус отображается:
- Цветная граница (2px) вокруг выпадающего списка
- Полупрозрачный фон цвета статуса (20% прозрачности)
- Текст выделен жирным шрифтом (font-weight: 500)

## Логика цветов

```javascript
const getStatusColor = (statusText) => {
  if (!statusText) return 'transparent';
  const lowerStatus = statusText.toLowerCase();
  if (lowerStatus.includes('процесс')) return '#3b82f6'; // синий
  if (lowerStatus.includes('отложен')) return '#eab308'; // желтый
  if (lowerStatus.includes('завершен')) return '#10b981'; // зеленый
  return '#6b7280'; // серый по умолчанию
};
```

## Как использовать

1. Убедитесь, что миграция `012_create_task_statuses.sql` выполнена
2. Статусы автоматически загружаются из базы данных
3. Выберите статус из выпадающего списка для задачи
4. Статус сохраняется автоматически с задержкой 500мс
5. Цвет статуса обновляется мгновенно в интерфейсе

## Порядок колонок в таблице

1. № - номер задачи
2. Приоритет - флаг приоритетности
3. Задача - название задачи
4. Ответственный - член команды
5. Дата постановки - дата создания
6. Дедлайн - срок выполнения
7. Примечание к выполнению - дополнительная информация
8. **Статус - новая колонка**
9. Выполнено - чекбокс завершения
10. Действия - кнопка удаления
