# Инструкция по миграции таблицы Tasks

## Файл миграции
`010_create_tasks_table.sql`

## Что делает миграция

1. **Удаляет старую таблицу tasks** (если существует) для чистой миграции
2. **Создает новую таблицу tasks** со следующими полями:
   - `id` - UUID, первичный ключ
   - `name` - название задачи (TEXT, обязательное)
   - `object_id` - ID объекта (ссылка на objects, CASCADE DELETE)
   - `team_member_id` - ID ответственного (ссылка на team_members, SET NULL при удалении)
   - `created_at` - дата и время постановки задачи
   - `deadline` - срок выполнения (DATE)
   - `note` - примечание к задаче
   - `is_completed` - чекбокс выполнения (BOOLEAN, по умолчанию FALSE)
   - `is_high_priority` - чекбокс приоритетности/красный флажок (BOOLEAN, по умолчанию FALSE)
   - `order_number` - порядковый номер задачи

3. **Создает индексы** для оптимизации запросов:
   - По object_id
   - По team_member_id
   - По deadline
   - По комбинации object_id + order_number
   - По is_completed
   - По is_high_priority

4. **Настраивает RLS политики** для полного доступа всем пользователям

## Как применить миграцию

### Вариант 1: Через Supabase Dashboard
1. Откройте Supabase Dashboard
2. Перейдите в SQL Editor
3. Скопируйте содержимое файла `010_create_tasks_table.sql`
4. Вставьте в редактор и нажмите "Run"

### Вариант 2: Через Supabase CLI
```bash
supabase db push
```

## Связанные изменения в коде

Файл `src/pages/TasksPage.jsx` был обновлен для работы с новой структурой таблицы:
- `title` → `name`
- `responsible_id` → `team_member_id`
- Убрана ссылка на поле `color` (которого нет в team_members)
- Обновлены запросы Supabase для корректной загрузки данных

## Проверка после миграции

После применения миграции проверьте:
1. Таблица `tasks` создана в Supabase
2. Все индексы созданы
3. RLS политики активны
4. Внешние ключи работают корректно
5. На странице `/objects/:id/tasks` можно добавлять и редактировать задачи
