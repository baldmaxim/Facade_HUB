# Автозаполнение ВОР по фасадным работам

Автоматическая расценка Ведомости Объёмов Работ (ВОР) для фасадных конструкций: НВФ, СПК, мокрый фасад, утепление, ограждения, двери.

## Как это работает

1. Пользователь загружает пустой ВОР заказчика (Excel)
2. Парсер извлекает позиции и примечания
3. Матчер подбирает фасадные шаблоны по ключевым словам (включая примечание заказчика)
4. Генератор строит заполненный Excel со всеми работами, материалами, коэффициентами и путями затрат
5. Основные материалы идут без цены (прайсинг позже из КП), вспомогательные — с заполненной ценой

```
src/lib/vorMatcher.js         → шаблоны фасадных работ + правила матчинга
src/lib/vorExcelGenerator.js  → парсинг и генерация Excel
src/pages/VorPage.jsx         → UI
```

## Шаблоны фасадных работ

**СПК (светопрозрачные конструкции):**
- `spk_profile` — профиль стойка-ригель
- `spk_glass` — заполнение стеклопакетом
- `spk_broneplenka` — бронирующая плёнка
- `spk_hardware` — фурнитура
- `doors_entrance` — входные двери (БКФН, витражные)
- `doors_tambour` — тамбурные двери (отдельный costPath)

**НВФ (навесной вентилируемый фасад):**
- `nvf_subsystem` — подсистема НВФ
- `insulation` — утепление (универсальная толщина, 2 слоя, мембрана всегда)
- `nvf_cladding_clinker` — клинкерная плитка (2 работы: облицовка + затирка FM.R-E)
- `nvf_cladding_cassette` — алюминиевые кассеты
- `nvf_cladding_fibrobeton` — стеклофибробетон
- `nvf_cladding_ceramic` — керамические панели
- `nvf_cladding_porcelain` — керамогранит
- `nvf_cladding_natural_stone` — натуральный камень
- `nvf_cladding_akp` — алюмокомпозитная панель
- `nvf_cladding_concrete_tile` — бетонная плитка
- `nvf_cladding_brick` — кирпич облицовочный
- `nvf_cladding_fcp` — фиброцементная панель
- `nvf_cladding_galvanized` — оцинкованный лист
- `nvf_cladding_arch_concrete` — архитектурный бетон
- `nvf_cladding_profiles_vertical` — вертикальные прямоугольные профили (2 работы: монтаж профилей + монтаж креплений)

**Мокрый фасад:**
- `wet_facade` — полный мокрый фасад ТЕХНОНИКОЛЬ (ROCKforce + ROCKglue + утеплитель inline + дюбель + ROCKmortar + сетка + ROCKprimer + штукатурка + ROCKsil)
- `wet_facade_insulation` — только клеевое утепление (fallback)
- `wet_facade_finish` — декоративный слой / штукатурка (fallback)
- `wet_facade_paint` — окраска

**Ограждения и козырьки:**
- `glass_railing` — стеклянные ограждения (EPDM базовый + EPDM установочный + анкерный крепёж Ø14х110)
- `glass_railing_molled` — моллированные ограждения (+ гибка профиля)
- `glass_canopy` — козырьки из триплекса

**Прочее:**
- `flashings` — отливы / откосы / парапеты (2 работы: изготовление оцинкованных элементов + устройство герметика СТИЗ-А/В)
- `vent_grilles` — вентиляционные решётки
- `scaffolding` — леса и люльки
- `kmd_spk`, `kmd_nvf` — разработка КМД
- `mockup` — мокап фасада

## Умная логика

**Автодетекция толщины утеплителя НВФ** по названию/примечанию:
- Паттерны: `NNN мм`, `толщ.NNN`, габариты `x NNN`
- Формула: наружный слой всегда **50 мм** (j=0.05), внутренний = (толщина−50)/1000
- При T ≤ 50 мм — один слой
- Поддержка формата `"X+Y мм"` — разбивка через `detectInsulationLayers()`
- Противопожарная мембрана ТехноНИКОЛЬ АЛЬФА ПРО НГ — **всегда присутствует**

**Утеплитель в мокром фасаде** — вставляется inline в генераторе между ROCKglue и дюбелем (не как отдельный шаблон-secondary). Толщина и тип берутся из названия позиции.

**Пропускаемые позиции:**
- Перголы — всегда ручная расценка
- Пол в технических лоджиях — не наш раздел

**Вторичные шаблоны** (scaffolding, КМД) добавляются автоматически к СПК/НВФ позициям. Если в ВОР есть отдельная позиция "Леса" или "Разработка КМД" — вторичный шаблон из других позиций **исключается** (нет дублей).

**Два стиля ВОР** — авто-детекция по маркеру "Прочие материалы":
- **Simple** (Сокольники, Событие): работы + материалы напрямую
- **Split-3** (Муза): позиции разбиты на роли (work / material / auxiliary)

## Приоритет правил матчинга

Специфичные правила — первыми, общие — последними:

1. Пропускаемые: освещение фасада, **перголы**, **пол в тех. лоджиях**
2. Мокап, фурнитура, КМД
3. Защитные экраны, ограждения, моллированные
4. **Откосы с мокрым фасадом** → `wet_facade + flashings`
5. **Откосы с кассетами** → `nvf_subsystem + cassette + flashings`
6. **Навесные на витраж / пилоны** → `nvf_subsystem + cassette` (без утепления)
7. **Короба кондиционеров** → `nvf_subsystem + cassette` (без утепления)
8. **Решётки под лоджиями** → полный НВФ с кассетами
9. **Вертикальные прямоугольные профили** → `nvf_cladding_profiles_vertical`
10. Тамбура, двери, окна — **до** стоечно-ригельн
11. НВФ по типу облицовки: клинкер, кассеты, фибробетон, керамика, камень...
12. **Полный мокрый фасад** (`wet_facade`) — **до** `wet_facade_insulation` и штукатур
13. Штукатур / мокрый фасад fallback → `wet_facade_finish`
14. Утеплитель, леса, подсистема, витраж — общие fallback

## Ограничения

- Цены основных материалов не заполняются — прайсинг из КП идёт отдельно
- Бренды в шаблонах — плейсхолдеры ("по проекту"), кроме фиксированных (затирка FM.R-E, дюбели с ценами)
- Двери в составе витража: в одних тендерах профиль учитывается в строке, в других — уже включён. Правится вручную.
