# Автозаполнение ВОР по фасадным работам

Автоматическая расценка Ведомости Объёмов Работ (ВОР) для фасадных конструкций: НВФ, СПК, мокрый фасад, утепление, ограждения, двери.

## Как это работает

1. Пользователь загружает пустой ВОР заказчика (Excel)
2. Парсер извлекает позиции и примечания
3. Матчер подбирает фасадные шаблоны по ключевым словам (включая примечание заказчика)
4. Генератор строит заполненный Excel со всеми работами, материалами, коэффициентами и путями затрат
5. Основные материалы идут без цены (прайсинг позже из КП), вспомогательные — с заполненной ценой

```
src/lib/vorMatcher.js         → шаблоны фасадных работ + правила матчинга
src/lib/vorExcelGenerator.js  → парсинг и генерация Excel
src/pages/VorPage.jsx         → UI
```

## Шаблоны фасадных работ

20+ шаблонов по типам конструкций:

**СПК (светопрозрачные конструкции):**
- `spk_profile` — профиль стойка-ригель
- `spk_glass` — заполнение стеклопакетом
- `spk_broneplenka` — бронирующая плёнка (защита СПК)
- `spk_hardware` — фурнитура
- `doors_entrance` — входные двери (БКФН, витражные)
- `doors_tambour` — тамбурные двери (разный costPath)

**НВФ (навесной фасад):**
- `nvf_subsystem` — подсистема НВФ + утеплитель
- `nvf_cladding_clinker` — клинкерная плитка
- `nvf_cladding_cassette` — алюминиевые кассеты
- `nvf_cladding_fibrobeton` — стеклофибробетон
- `insulation` — утепление НВФ (универсальная толщина)

**Мокрый фасад (3 стадии):**
- `wet_facade_insulation` — клеевое утепление
- `wet_facade_finish` — декоративный слой / штукатурка
- `wet_facade_paint` — окраска

**Прочее:**
- `glass_railing`, `glass_railing_molled` — стеклянные ограждения (прямые и моллированные)
- `glass_canopy` — козырьки
- `flashings` — отливы/откосы/парапеты
- `vent_grilles` — вентиляционные решётки
- `scaffolding` — леса
- `kmd_spk`, `kmd_nvf` — разработка КМД
- `mockup` — мокап

## Два стиля ВОР

Авто-детекция стиля по маркеру "Прочие материалы":

**Simple** (Сокольники): одна позиция = все шаблоны (работы + материалы) напрямую

**Split-3** (Муза): позиции разбиты на роли
- `work` — вставляем только работы
- `material` — вставляем только основные материалы
- `auxiliary` ("Прочие материалы") — вспомогательные со всего кластера

**Гибридный режим:** в одном split-3 файле часть позиций может быть standalone (без последующего "Прочие материалы"). Они обрабатываются по simple-логике автоматически через `isClusteredWork`/`isClusteredMaterial`.

## Умная логика

**Автодетекция толщины утеплителя** по названию/примечанию:
- Паттерны: `NNN мм`, `толщ.NNN`, габариты `x NNN`
- Универсальная формула: при T > 100мм — 2 слоя (внутренний 100мм, наружный (T-100)мм), при T ≤ 100 — один слой
- Применяется per-position или per-cluster

**Вторичные шаблоны** (scaffolding, КМД, фурнитура) добавляются автоматически ко всем СПК/НВФ позициям. Но если в ВОР есть **отдельная позиция** (например "Леса и люльки" или "Разработка КМД СПК"), вторичный шаблон из других позиций **исключается** — чтобы не было дублей.

**Замена шаблонов в кластере:** например, если в split-3 кластере первая работа — `wet_facade_insulation`, то материальная строка утеплителя в этом кластере автоматически использует `wet_facade_insulation` вместо НВФ `insulation`.

## Приоритет правил матчинга

Порядок правил критичен — специфичные правила идут первыми:

1. Мокап, освещение (пропускаемые)
2. Фурнитура, КМД (отдельные позиции)
3. Профиль, защита, молл, ограждения
4. Тамбура, двери входные, окна — **до** "стоечно-ригельн"
5. НВФ: фибробетон, клинкер, кассета — **до** общего "стекл"
6. Wet facade (отделка, штукатур, краска) — **до** общего "утеплен"
7. Fallback-правила (витраж, клинкер, кассет)

## Ограничения

- Цены основных материалов не заполняются автоматически — стадия прайсинга из КП идёт отдельно
- Бренды/конкретные продукты в шаблонах — плейсхолдеры ("по проекту")
- Сложные случаи (моллированные без ключевого слова, тамбур без явного указания) корректируются вручную
